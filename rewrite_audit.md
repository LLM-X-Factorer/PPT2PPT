# Action Title 重写审计 · v0

**目标**：把"结构标签式标题"（产品矩阵 / 当前问题 / 年度 OKR）改写成"结论式标题"，让汇报对象一眼看到判断。

**改动范围**：仅 title / subtitle 文本层。所有 items / rows / KRs / 甘特任务名 **未动**——OKR 是承诺，甘特是事实，砍这些会失真。

**改动哲学**：
- 标题 = 这一页的"如果只能记住一句话"
- 副标题 = 承接结论的关键时间锚点或数字
- 不在标题里堆名词；让动词、判断、数字进来

## 没改的页面

- **SECTION_PRODUCT / SECTION_BRAND**：章节大字页本身就是叙事节拍（让听众喘口气），不该塞结论。
- **CLOSING**："把每一件事做到位"已经是 action title——这本来就是结尾页该有的样子，不动。
- **所有 KRs / items 内文 / 甘特任务名**：这些是承诺/事实，是汇报的骨架。第一版只重写"标题层"。

## 已知争议点

1. **PRODUCT_Q2 "三件事打透" vs 实际 5 group**：标题做减法 = 主观裁剪。如果汇报对象是细节型领导，可能会问"为什么不提沙龙和周边"——这种时候要不要接受"减法的代价"，是项目要回答的核心问题。
2. **BRAND_PROBLEMS 标题尖锐度**："传播没方法论"在内部汇报里是诚实的自评，但对上汇报或者对外发布，可能需要留余地。Action Title 的"力度"应该可调。
3. **数字硬编码进标题**（"9 个账号 / 4 跑款"）：每次数据变都要重写标题。如果工具化，需要考虑标题模板 + 数据动态填充。

这三点是 v1 工具化时要解决的设计问题。
